# 仲裁智能化项目 v1 UI 规范

## 1. 总体风格

- 产品类型：仲裁委 B 端工作台
- 目标气质：专业、可信、克制、清晰
- 关键词：浅底分层、权威蓝、真实业务感、结构优先
- 禁止方向：泛 AI 风格、黑底科技感大屏、抽象炫光界面、脱离仲裁委语境的互联网后台
- 不依赖 Element Plus，使用自建纯 CSS 设计系统

## 2. 色彩策略

- 主品牌色：深权威蓝（#2f5cf5）
- 主品牌色浅色：浅蓝高亮（#ebf2ff）
- 页面背景色：浅灰蓝近白色（#f3f6fb）
- 卡片背景色：纯白
- 边框色：冷灰蓝边框（#dce5f2）
- 文本主色：深灰蓝（#344054）
- 文本次色：中性灰蓝（#6b778c）
- 成功色：绿色（#10B981 / #1f9d66）
- 预警色：橙色（#F59E0B / #d9902f）
- 风险色：红色（#EF4444 / #e25555）

规则：
- 大面积背景保持浅色，避免压迫感
- 主色用于标题头、主按钮、关键状态，不要全页滥用
- 风险色只服务业务状态，不做装饰

## 3. 字体与排版

- 正文字体：系统无衬线，强调可读性
- 标题层级控制在 3 级以内
- 字重以 600 / 700 为主，保持克制
- 文书编辑器内文本 line-height: 1.8，标题居中，段首缩进 2em

## 4. 布局原则

- 默认工作台全宽三栏布局（左侧导航/材料 + 中部主内容 + 右侧辅助/AI）
- 编辑器中区使用白纸样式（.wb-paper：max-width 800px、居中、白底、圆角、阴影）
- 首屏必须回答"这个案子在争什么"
- 高密度信息通过分组和层次解决，不靠缩小字号硬塞

## 5. 组件规范

### 按钮
- 主按钮：蓝色，数量严格控制
- 次按钮：白底描边
- 危险按钮：只用于高风险动作

### 卡片
- 默认卡片：白底、浅边框（.wb-card）
- 高亮卡片：边框变色 + 浅底强调
- 风险卡片：浅红底或左色条

### 浮动工具栏
- 选中文本后出现在选区上方
- 白底圆角阴影，图标+文字的横排按钮
- 包含 5 个 AI 操作（续写/改写/扩写/精简/总结）和批注

### 批注（飞书文档风格）
- 从选中文本发起，不从右侧面板发起
- 卡片顶部展示选中原文引用（黄色竖线+文本）
- hover 显示操作图标（编辑/已解决/删除）
- 点击卡片自动定位正文
- 状态：待处理（黄点）/ 已解决（绿点）

### 手风琴式材料面板（文书写作助手左栏）
- 每个材料项点击展开/收起其下证据条目
- 材料标题行显示：文件名、提交方标签（彩色）、证据条数、已引用数
- 顶部提交方筛选（全部/申请方/被申请方/仲裁庭）
- 已引用证据显示绿色"已引用"标记

### 模块导航条（编辑器顶部）
- 工具栏下方横向标签：首部·仲裁请求·案件事实·答辩·查明·认为·裁决主文
- 点击跳转，滚动跟随自动高亮当前模块
- 当前模块蓝底白字，其他模块灰色

### 校对检查侧边标记
- 有问题的段落左侧显示 3px 竖线（红色=error，橙色=warning）
- 不需要切到校对 Tab 也能感知问题位置

### Toast 消息
- 全局顶部居中浮出
- 类型：success（绿）/ error（红）/ warning（橙）/ info（蓝）
- 2.5 秒自动消失

### 自动保存指示器
- 编辑器工具栏右侧
- 绿色文字 + 勾选图标 + "已自动保存 HH:MM"

## 6. 状态表达

- 成功：绿色标签或浅绿边框
- 关注：橙色标签或浅橙边框
- 风险：红色标签或浅红提示
- 缺失：灰红色提示
- 禁用：灰色文本和灰底

规则：状态必须同时通过颜色和文字表达

## 7. 动效原则

- 只做进入、切换、定位、高亮反馈
- 校对检查/批注定位使用 scrollIntoView + 2s 黄色渐隐高亮
- 手风琴展开使用 slideDown 动画
- Toast 使用 slide + fade 进出
- 装饰必须让位于信息清晰度

## 8. 实现文件

- 设计变量：`src/styles/design-tokens.css`
- 通用组件原语：`src/styles/workbench.css`（.wb-paper、.wb-card、.wb-btn、.wb-tabs 等）
- 仲裁业务样式：`src/styles/arbitration-kit.css`
- 编辑器样式：`src/components/editor/EditorArea.vue`（scoped + :deep）
- Toast 组件：`src/components/ToastMessage.vue`
- 不建议组件内大段行内样式，尽量收进组件样式或公共样式层

## 9. 参考基线

- 参考项目：[Arbitration_panel](https://github.com/en175/Arbitration_panel)
- 明确继承：浅底工作台、权威蓝、卡片分层、克制高端感
- 明确不继承：不照搬具体业务页面，不扩散玻璃效果
